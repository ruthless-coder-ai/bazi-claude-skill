# bazi — 八字排盘与解读 Skill

一个给AI用的八字（四柱）排盘与命理解读 skill：排盘交给代码，解读交给模型，各干各的擅长事。

## 用起来是什么样（30 秒）

> **你**：帮我算下八字，1990 年 3 月 15 日下午两点半生的，北京，女。
>
> **Claude**：跑排盘脚本（节气精确到分钟、真太阳时校正 −25 分钟），摆出四柱表、大运表、五行统计，给一段三五句的总纲，然后**停下来问**：
>
> 「想先看哪一块——事业、感情、近几年运势？或者你最近具体在纠结什么，直接说。也可以先拿两三件过去的大事（哪年换的工作、哪年结的婚）来对对盘，看看这套读法在你身上灵不灵。」

之后随便追问：明年运势、什么时候适合跳槽、配偶宫怎么看。聊完可以让它把整场解读存成一份 markdown 报告。

## 解读长什么样（真实示例摘录）

上面那个生日真实跑出来的盘（示例数据，非真人）：

**盘面**（只放可复核的事实）：

| | 年柱 | 月柱 | 日柱 | 时柱 |
|---|---|---|---|---|
| 天干 | 庚（伤官） | 己（比肩） | **己（日主）** | 辛（食神） |
| 地支 | 午 | 卯 | 卯 | 未 |

五行：木 2、火 1、土 3、金 2、**水 0**　·　桃花两见（月、日）　·　真太阳时 14:30 → 14:05

**解读**（摘录，注意每句都带着推理链和置信口径）：

> 你日主己土，生在卯月——春天木当令，土不得令【盘面】。但你不是孤军：天干两个己土并肩，时支未土给你本气根【盘面】→ 按传统读法，这算失令而有帮，身弱但没弱到底【框架内的标准判法】→ 所以火土的大运流年通常更托你，木金重的年份更耗你【推断】。月支日支两处卯木都是七杀藏在暗处——压力对你不是偶发事件，是常驻背景。这股力量没有印星明透来化【盘面】，课题不是躲压力，是给压力找一个出口……往细里说就见仁见智了，桃花两见的讲法，你可以拿自己的经历来检验。

「盘面」区拿去任何排盘工具都能复核；「解读」区每个判断露出推导，你不同意可以指着具体一环反驳——这是这个 skill 和"一口断"式算命的根本区别。

## 你需要准备什么

- **出生日期**（公历农历都行，农历会确认是否闰月）
- **出生时间**，尽量精确到分——真太阳时校正后差半小时可能换时柱
- **性别**（大运顺逆的排法需要）
- **出生城市**（做真太阳时校正；海外出生支持任意时区和夏令时）

不知道时辰？照样排三柱，skill 会明说哪些结论（子女宫、晚年运）精度受限，绝不硬猜。

## 安装

前提：[Claude Code](https://claude.com/claude-code)（终端/桌面版均可）+ Python 3。

```bash
git clone https://github.com/ruthless-coder-ai/bazi-claude-skill.git ~/.claude/skills/bazi
pip install lunar-python
```

装完跑一条命令验证（顺便看看排盘输出长什么样）：

```bash
python3 ~/.claude/skills/bazi/scripts/paipan.py --date 1990-03-15 --time 14:30 --gender 女 --longitude 116.4
```

打印出 JSON 排盘即就绪。若报缺 `lunar-python`，多半是 pip 和 python3 不在同一环境（conda 用户常见），用 `python3 -m pip install lunar-python` 最稳。

**隐私**：排盘计算完全在你本机跑；出生信息只存在于你和 Claude 的对话里，不会发给任何其他第三方。

## 为什么这个 skill 靠谱

### 排盘引擎（算得对）

**1. 干支从不心算。** 大语言模型心算干支历是重灾区——节气边界错一小时，年柱月柱全盘皆错。四柱、藏干、十神、大运、流年、流月、神煞全部由 `scripts/paipan.py` 计算（基于 [lunar-python](https://github.com/6tail/lunar-python)），模型只负责读盘和解读。

**2. 节气精确到分钟。** 换年柱看立春的精确时刻，换月柱看交节的精确时刻。实测案例：1990-02-04 立春在当天 10:14，上午 9 点出生仍是己巳年丁丑月——按天粗算的排盘器在这里会错。结果与独立实现的 sxtwl（寿星天文历）交叉验证过。

**3. 真太阳时校正。** 按出生城市经度差 + 均时差校正时柱；海外出生支持任意时区和夏令时。钟表时间和太阳时可以差出半小时以上，足以改变时柱。

**4. 临界情形主动报警，不装确定。** 距节气交接 48 小时内、真太阳时距时辰边界 15 分钟内、晚子时（两派日柱不同，明示所用流派）、校正后跨日——脚本都会标出，两可时排两个盘对照定夺。

**5. 信息不全时优雅降级。** 不知时辰排三柱并明说局限；没有性别就不排大运，绝不硬猜。

### 解读方法论（说得诚实）

**6. 完整的子平法推理链，不是玄学随机输出。** `references/` 把方法论固化成可执行步骤：日主强弱（得令得地得势）→ 格局 → 用神喜忌 → 宫位 → 大运流年应事扳机，外加分类解答手册。

**7. 确定性分层，在用户眼前分。** 输出强制两区：「盘面」区只放脚本可追溯的事实，「解读」区每个判断露出推理链，置信度三级（计算事实 / 框架内强推断 / 一家之言）标在行文里。没有盖住全文的万能免责句——诚实做在每一条推断里，逐条可辩驳。

**8. 问题优先 + 历史校准闭环。** 总纲之后先停下来问你想看哪块，不倒全景宣判；主动邀请你拿过去的大事年份对账——对上的说机制，对不上的明说、降置信度，绝不硬圆。时辰两可的盘正好借这一步定盘。

**9. 有声音，且防鸡汤。** `references/voice.md` 把"怎么说"写成可执行规则：判词一律翻成镜子（"伤官见官易犯小人"❌ → "你的锋芒和规则贴得很近，课题是找到容得下它的场"✅）；每句话必须锚定本盘具体的字，放到别人盘上多数不成立才算合格——巴纳姆万能句、空洞升华、伪深刻排比、廉价共情全部明令禁止并配负例。

**10. 凶信息走决策程序，不吓人也不隐瞒。** 三分支：用户问了且可准备且证据强→直说；临近且证据强→转成可准备的课题；远期证据弱→这次先不提（但直接问到永远诚实答）。断死期、判绝症类字眼任何分支都禁用。

**11. 两种语调可选。** 冷静智性陪伴（默认）vs 江湖通透，可点名切换。只改说法不改内容：同样的盘面依据、同样的置信度、同样的克制。

## 能做什么，不做什么

单人盘的全部话题（性格格局、人生阶段、婚姻感情、事业财运、六亲子女、健康、流年流月）都做。边界如下，被问到时会明说而不是硬答：

- **合盘**：两人生辰都给才做简批（各自感情模式 + 日柱互动），明确不是传统合婚，不下"合不合"断语
- **择日、起名打分、风水**：不做（各是独立手艺），只给喜用五行/顺月这类本盘能给的参考
- **转运物**：永远不推荐买任何东西——盘上能动的杠杆在选择、时机、习惯
- **窥探他人、彩票赌博**：不做

## 脚本单独使用

排盘脚本不依赖 Claude，可独立运行：

```bash
python3 scripts/paipan.py --date 1990-03-15 --time 14:30 --gender 女 --longitude 116.4
python3 scripts/paipan.py --date 1990-02-19 --lunar --gender 男     # 农历输入
python3 scripts/paipan.py --date 1995-08-20 --time 15:00 --gender 女 \
  --longitude 13.4 --tz-meridian 30                                  # 柏林出生，夏令时 UTC+2
```

输出为结构化 JSON：四柱（含藏干十神纳音）、五行统计、空亡、神煞、起运信息、各步大运、当前大运下的流年与当年十二流月，以及 `提醒` 数组（所有临界警告）。

## 目录结构

```
bazi/
├── SKILL.md              # skill 入口：触发条件 + 工作流程 + 边界
├── scripts/paipan.py     # 排盘脚本（输出 JSON）
└── references/
    ├── fundamentals.md   # 基础知识：排盘结果的读法
    ├── analysis.md       # 分析方法论 + 两区输出/置信分层 + 历史校准
    ├── topics.md         # 分类解答手册
    └── voice.md          # 声音：判词→镜子、反巴纳姆清单、凶信息决策程序、双语调
```
