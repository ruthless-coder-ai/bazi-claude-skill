# bazi — 八字排盘与解读 Skill

一个给AI用的八字（四柱）排盘与命理解读 skill：排盘交给代码，解读交给模型，各干各的擅长事。

对话里说「帮我算下八字」「1990年3月15日下午2点生的，看看我的命」「明年运势如何」即可触发；支持追问婚姻、事业财运、早/青/中/晚年各阶段、六亲子女、健康，以及精确到流月的近期运势。

## 为什么这个 skill 靠谱

**1. 干支从不心算。** 大语言模型心算干支历是重灾区——节气边界错一小时，年柱月柱全盘皆错。这个 skill 的四柱、藏干、十神、大运、流年、流月、神煞全部由 `scripts/paipan.py` 计算（基于 [lunar-python](https://github.com/6tail/lunar-python)），模型只负责读盘和解读。

**2. 节气精确到分钟。** 换年柱看立春的精确时刻，换月柱看交节的精确时刻，而不是按"哪一天"粗算。实测案例：1990-02-04 立春在当天 10:14，上午 9 点出生仍是己巳年丁丑月——按天粗算的排盘器在这里会给出错误的庚午年。排盘结果与独立实现的 sxtwl（寿星天文历）做过交叉验证。

**3. 真太阳时校正。** 排盘前询问出生城市，按经度差 + 均时差校正时柱；海外出生支持任意时区和夏令时（`--tz-meridian`）。钟表时间和太阳时可以差出半小时以上，足以改变时柱。

**4. 临界情形主动报警，不装确定。** 脚本会检测并提醒：

- 出生距节气交接 48 小时内（年/月柱临界，建议核对出生时间）
- 真太阳时距时辰边界 15 分钟内（时柱两可，出两个盘对照）
- 晚子时出生（23:00–01:00，明示所用流派，另一流派日柱不同）
- 校正后跨日（日柱可能变）

**5. 信息不全时优雅降级。** 不知道时辰就排三柱并明说子女宫/晚年运精度受限；没有性别就不排大运（大运顺逆依年干阴阳+性别），绝不硬猜。

**6. 完整的解读方法论，不是玄学随机输出。** `references/` 三件套把子平法固化成可执行的推理链：

- `fundamentals.md` — 干支五行、藏干、十神白话含义、合冲刑害、常用神煞
- `analysis.md` — 定盘顺序：日主强弱（得令得地得势）→ 格局 → 用神喜忌 → 宫位 → 大运流年应事扳机
- `topics.md` — 分类解答手册：每类问题看哪里、答案怎么组织，含流年十神速查表

**7. 断语有分寸。** 内置输出规范：说倾向不说定数，凶不吓人、吉不打包票；健康婚姻等敏感话题先回应情绪、给建议不给恐吓；重大决定明确交还用户。每次完整解读附一句"供参考与自我觉察"的说明。

**8. 传统术语 + 现代白话双轨。** 分析框架是正统子平法（格局用神十神），输出翻译成现代人能懂的话——正官是"规则与职位"，七杀是"压力与竞争之星"，术语首次出现必带解释。

## 安装

```bash
git clone https://github.com/ruthless-coder-ai/bazi-claude-skill.git ~/.claude/skills/bazi
pip install lunar-python
```

然后在 Claude Code 里直接问八字即可（或 `/bazi` 触发）。

## 脚本单独使用

排盘脚本不依赖 Claude，可独立运行：

```bash
python3 scripts/paipan.py --date 1990-03-15 --time 14:30 --gender 女 --longitude 116.4
python3 scripts/paipan.py --date 1990-02-19 --lunar --gender 男     # 农历输入
python3 scripts/paipan.py --date 1995-08-20 --time 15:00 --gender 女 \
  --longitude 13.4 --tz-meridian 30                                  # 柏林出生，夏令时 UTC+2
```

输出为结构化 JSON：四柱（含藏干十神纳音）、五行统计、空亡、神煞、起运信息、各步大运、当前大运下的流年与当年十二流月，以及 `提醒` 数组（所有临界警告）。

## 目录结构

```
bazi/
├── SKILL.md              # skill 入口：触发条件 + 工作流程
├── scripts/paipan.py     # 排盘脚本（输出 JSON）
└── references/
    ├── fundamentals.md   # 基础知识：排盘结果的读法
    ├── analysis.md       # 分析方法论 + 断语分寸
    └── topics.md         # 分类解答手册
```
