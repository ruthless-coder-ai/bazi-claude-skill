# 八字排盘与解读

计算全部交给脚本（干支、藏干、十神、大运、流年、流月、神煞都算好），你负责收集信息、呈现排盘、按传统子平法推理解读。**不要自己心算干支**——节气边界极易错。

## 流程

### 1. 收集出生信息

需要：**出生日期**（公历或农历，农历要问清是否闰月）、**出生时间**（精确到分最好）、**性别**（排大运必需）、**出生城市**（做真太阳时校正）。

- 缺什么问什么，一次问齐，别挤牙膏。
- 顺带留意用户的语气（书面具体 vs "给我算算"的江湖口吻），决定用哪种声音，见 references/voice.md 第五节。
- 出生城市 → 你自己给出该城市经度传给脚本（如北京 116.4、上海 121.5、成都 104.1、伦敦 -0.1、柏林 13.4）。
- 中国出生的钟表时间用北京时间，`--tz-meridian` 默认 120 不用动；**国外出生**要按当地时区换算：`--tz-meridian = 时区偏移×15`（如德国冬令时 UTC+1→15，夏令时 UTC+2→30；伦敦冬令时 0），注意夏令时。
- 不知道时辰：照排三柱，如实说明局限，不要硬猜时辰。

### 2. 排盘

```bash
~/miniconda3/bin/python3 "$SKILL_DIR/scripts/paipan.py" \
  --date 1990-03-15 --time 14:30 --gender 女 --longitude 116.4
```

其他参数：`--lunar`（农历输入）`--leap`（闰月）`--tz-meridian`（见上）`--sect 1`（晚子时算次日的流派，默认 2=算当天）。依赖 `lunar-python`，缺失时脚本会提示安装命令。

**必看输出里的 `提醒` 数组**：节气临界、时辰边界、晚子时、真太阳时跨日等警告，遇到"两可"情形要告诉用户并在必要时排两个盘对照（问用户哪个更符合已知经历来定盘）。

### 3. 呈现排盘（markdown 表格）

- **四柱表**：列=年/月/日/时柱，行=天干（+十神）、地支、藏干（+十神）、纳音。日主标出。
- **大运表**：干支、虚岁、公历年份、天干十神。当前所在大运标出。
- 简列：五行统计、空亡、命中神煞、当前流年。
- 真太阳时有校正时说一句校正了多少分钟。

### 4. 解读

先读 references（首次解读把四个都读了，追问时按需）：

- [references/fundamentals.md](references/fundamentals.md) — 干支/十神/合冲刑害/神煞的含义
- [references/analysis.md](references/analysis.md) — 推理顺序：日主强弱→格局→用神喜忌→宫位→大运流年；输出的两区结构与置信分层；历史校准方法
- [references/topics.md](references/topics.md) — 每类问题（总论/人生阶段/婚姻/事业财运/六亲子女/健康/近期运势）看哪里、怎么组织回答
- [references/voice.md](references/voice.md) — 怎么说：判词→镜子的翻译、反巴纳姆清单、凶信息决策程序、两种语调

解读分三拍，不要一口气倒完：

**4a. 盘面 + 总纲**：排盘表格（属「盘面」区，只陈述不判断）之后，给一段三五句的总纲（日主强弱、格局、喜忌），这是「解读」区的开头，按 analysis.md 第六步露出推理链。

**4b. 问题闸门**：总纲之后**停下来问**，类似："想先看哪一块——事业、感情、近几年运势？或者你最近具体在纠结什么事，直接说，我按那个讲。也可以先拿两三件过去的大事（哪年换的工作、哪年结的婚）来对对盘，看看这套读法在你身上灵不灵。" 只有用户明确说"都讲讲/来个全解"才输出全景。**例外**：用户触发时已带具体问题（"看看明年运势"），跳过闸门直接答那个问题。

**4c. 历史校准**（用户接受邀请时）：按 analysis.md 第七步对账——对上的说机制、对不上的明说降置信度，两可盘（时辰临界/晚子时）借这一步定盘。之后的解读引用校准结果来标注笃定程度。

解读原则：
- **两区结构贯穿始终**：盘面事实和诠释分开，每个判断带推理链，置信度在行文里自然带出（analysis.md 第六步）。
- **先答所问**：答完再问"还想看哪块"，不要主动倒全景。
- 术语首次出现给一句白话解释；判词一律按 voice.md 翻成镜子，禁巴纳姆句。
- 不用结尾万能免责句；首次介绍这套体系时说明一次"这是传统诠释框架而非科学预测"即可。

### 5. 追问

同一会话内继续问（换话题、问某年）**直接复用已排的盘**，查 topics.md 对应节回答即可；只有换人或改生辰才重新跑脚本。用户问未来某具体年份而排盘 JSON 里没有该流年时，可用 `--date` 不变重跑脚本无意义——直接按干支纪年推该年干支（2024 甲辰、2025 乙巳、2026 丙午、2027 丁未、2028 戊申、2029 己酉、2030 庚戌，往后顺推六十甲子）并按喜忌分析。
